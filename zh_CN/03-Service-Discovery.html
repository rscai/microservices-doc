理解服务注册与发现「Service Registry & Discovery」

服务注册与发现「Service Registry and Discovery」（有时穪为服务注册「Service Registry」或服务发现「Service Discovery」是微服务架构中常用的模式。

由于微服务架构的固有特性，生产环境中的服务都是不断地变化的，服务实例也是不断地被创建和销毁的。消费者服务需要一种机制去实时地发现可用的服务实例，服务注册与发现就是其中一种解决方案。

服务注册维护着服务实例的元数据（包含服务名穪、实例的主机地址、监听的端口等）。服务实例启动时主动向服务注册注册自身元数据，在停止时主动向服务注册注销自身元数据。

消费者服务通过服务名穪从服务注册获取可用的服务实例信息，然后直接与服务实例通信。

举个例子，Service Registry & Discovery 服务是一直存活着的，微服务的实例 instanceA 和 instanceB 存活的时间较短且其启动与停止皆是不可预知的。

  1. instanceA 启动之后会主动向 Service Registry & Discovery 注册自身；

  2. Consumer 先从 Service Registry & Discovery 发现（discover）微服务的实例；

  3. Service Registry & Discovery 从自已维謢的服务实例元信息中寻找一个可用的服务实例，将其IP及端口等信息返回给 Consumer；

  4. Consumer 通过服务实例的IP及端口访问服务实例；

  5. instanceA 在常规停止时可以将自身从 Service Registry & Discovery 处注销，但大多数情况下服务实例是被强制停止的，来不及主动注销，所以 Service Registry & Discovery 要有能力处理已死但未注销的服务实例；

  6. 新的 instanceB 启动时也会向 Service Registry & Discovery 注册；

  7. Consumer 再一次从 Service Registry & Discovery 发现服务实例；

  8. Service Registry & Discovery 从自己维护的服务实例元信息中寻找一个可用的服务实例 instanceB，将其 IP 及端口等信息返回给 Consumer；

  9. Consumer 通过服务实例的 IP 及端口访问服务实例。

理解Eureka

Eureka是由Netflix捐赠给开源社区的基于Spring Boot的服务注册与发现实现。

Eureka服务端提供服务注册服务，其没有将服务元数据持久化至磁盘。而且也没有必要将服务元数据持久化，因为服务实例的状态是随时变化的，持久化的状态毫无意义。

Eureka服务端提供了单独「Standalone」和结对「Peer」两种运行模式。默认情况下，Eureka服务端运行在结对模式。以下Spring Boot应用配置文件配置一组两节点结对的Eureka服务端：

application.yml
spring:
  profiles: peer1
eureka:
  instance:
    hostname: peer1
  client:
    serviceUrl:
      defaultZone: https://peer2/eureka/
---
spring:
  profiles: peer2
eureka:
  instance:
    hostname: peer2
  client:
    serviceUrl:
      defaultZone: https://peer1/eureka/

当Eureka服务端以profile peer1 运行在结点 peer1 时，将自己做为客户端注册至 peer2；当Eureka服务端以profile peer2 运行在结点 peer2 时，将自己做为客户端注册至 peer1。peer1 和 peer2 实时同步注册务元数据，当其中一个实例重启时，会从另一个实例同步元数据，从而实现高可用性「High Availability」。

结对模式也可用于多于两个Eureka服务端实例结对。以下Spring Boot应用配置文件配置了一组三节点Eureka服务端结对：

application.yml
eureka:
  client:
    serviceUrl:
      defaultZone: https://peer1/eureka/,https://peer2/eureka,https://peer3/eureka
---
spring:
  profiles: peer1
eureka:
  instance:
    hostname: peer1

---
spring:
  profiles: peer2
eureka:
  instance:
    hostname: peer2

---
spring:
  profiles: peer3
eureka:
  instance:
    hostname: peer3

当以profile peer1、peer2 和 peer3 分别运行三个Eureka服务端实例在三个主机上时，它们各自将自身注册至三个Eureka服务端。任一实例都实时与其它两个实例同步元数据。

Eureka服务端默认以结对模式运行，若要以单独「Standalone」模式运行，需显示关闭其客户端注册特性。

application.yml
eureka:
  instance:
    hostname: localhost
  client:
    registerWithEureka: false (1)
    fetchRegistry: false (2)
    serviceUrl:
      defaultZone: http://${eureka.instance.hostname}:${server.port}/eureka/ (3)
1 显式关闭服务注册。
2 显式关闭从对方同步服务注册元数据。
3 即使以单独模式运行，依旧要显式声明服务諯（即自身）URL。

协议

Eureka服务端开放RESTFul风格的Web服务，客户端和结对的其它服务端可以通过此RESTFul Web服务注册和发现服务。Eureka维护的核心元数据是应用「application」和实例「instance」。

应用「application」仅维护一个属性 appName。

实例「instance」以 InstanceInfo 的形式维护。InstanceInfo 维护着很多属性，其中服务消费者最常用的有：

  • ipAddr，服务实例的 IP 地址；

  • port，服务实例监听的端口；

  • securePort，服务实例监听的安全端口（HTTPS 服务监听的端口）。

服务实例心跳状态

服务实例应定时向服务端发送心跳状态。心跳状态通过 PUT 请求发送给服务端。心跳请求 URL 格式为 /eureka/apps/{appName}/{instanceId}。

心跳状态请求样例
PUT /eureka/apps/CATALOG/macbook-pro.lan:catalog:8081?status=UP&lastDirtyTimestamp=1576139630424 HTTP/1.1
DiscoveryIdentity-Name: DefaultClient
DiscoveryIdentity-Version: 1.4
DiscoveryIdentity-Id: 192.168.199.141
Accept-Encoding: gzip
User-Agent: Java-EurekaClient/v1.9.13

注销服务

客户端通 DELETE 请求注销服务实例，请求的目标URL格式为 /eureka/apps/{appName}/{instanceId}。

注销服务实例请求样例
DELETE /eureka/apps/CATALOG/macbook-pro.lan:catalog:8081 HTTP/1.1
DiscoveryIdentity-Name: DefaultClient
DiscoveryIdentity-Version: 1.4
DiscoveryIdentity-Id: 192.168.199.141
Accept-Encoding: gzip
User-Agent: Java-EurekaClient/v1.9.13

实现

借由 Spring Cloud Netflix，可以轻松地实现基于 Eureka 的服务注册与发现。

最简单的创建一个新的 Spring Boot 应用方法是使用 Spring Initializr。使用浏览器访问 https://start.spring.io :

  • Project 选择 Gradle Project，使用 Gradle 构建我们的项目；

  • Language 选择 Java；

  • Spring Boot 撰择 2.2.7 或其它 2.2.x 版本，最新版（讫今为止）的 Spring Cloud Hoxton 兼容 Spring Boot 2.2.x；

  • Project Metadata 部份，Group 填 io.github.rscai.microservices，Artifact 填 eureka，Name 填 eureka，Packaging 撰 Jar，Java 撰择版本 8；

  • Dependencies 选择

    • Eureka Server

单击GENERATE，Spring initializr 就会生成 ZIP 格式的项目文件。

将 eureka.zip 解压后得到一个 Spring Boot 项目。其中：

.
├── .gitignore
├── HELP.md
├── build.gradle
├── gradle
│   └── wrapper
│       ├── gradle-wrapper.jar
│       └── gradle-wrapper.properties
├── gradlew

    ├── main

  • .gitignore, 羅列了應被 GIT 忽略的文件和目錄。自動生成及臨時文件及目錄不應被納入版本控制；

  • build.gradle，Gradle 項目文件，定義了項結構及構建過程；

  • gradle/wrapper/ 包含所有 Gradle Wrapper 相關的文件；

  • gradlew 是 UNIX-like 平台版本的 Gradle 命令包裝；

  • gradlew.bat 是 Windows 平台版本的 Gradle 命令包裝；

  • HELP.md 顧名思義是幫助文檔，與 README 類似；

  • src/ 包含所有源代碼和資源文件：

  • src/main/ 包含所有部署至運行環境的源代碼文件（以編譯後的目標代碼形式被部署至運行環境）和資源文件：

  • src/main/java/ 包含所有部署至運行環境的 Java 源代碼文件：

  • src/main/resources/ 包含所有部署至運行環境的資源文件；

  • src/test 包含所有用於測試的源代碼和資源文件；

  • src/test/java/ 包含所有用於測試的 Java 源代碼文件；

  • src/test/resource/ 包含所有用於測試的資源文件。

将 eureka 项目导入 IntelliJ IDEA。首先，打开 Intellij IDEA，然后，单系 Import Project，选定 eureka 项目目录，再然后选择以 Gradle 项目形式导入。IntelliJ IDEA 会自动初始执行 Gradle 配置，下载 /gradle/wrapper/gradle-wrapper.properties 中指定版本的 Gradle。

创建Eureka服务端

首先，创建一个独立的Spring Boot应用实现Eureka服务端。

然后，配置 build.gradle。

build.gradle
plugins {
	id 'org.springframework.boot' version '2.1.0.RELEASE' (1)
	id 'io.spring.dependency-management' version '1.0.8.RELEASE' (2)
	id 'java'
}

group = 'io.github.rscai.microservices'
version = '0.0.1-SNAPSHOT'
sourceCompatibility = '1.8'

repositories {
	mavenCentral()
	maven { url 'https://repo.spring.io/milestone' } (3)
}

ext {
	set('springCloudVersion', "Greenwich.SR3") (4)
}

dependencies {
	implementation 'org.springframework.cloud:spring-cloud-starter-netflix-eureka-server' (5)
	testImplementation('org.springframework.boot:spring-boot-starter-test') {
		exclude group: 'org.junit.vintage', module: 'junit-vintage-engine'
	}
	testImplementation 'org.junit.jupiter:junit-jupiter-api'
	testRuntimeOnly 'org.junit.jupiter:junit-jupiter-engine'
}

dependencyManagement {
	imports {
		mavenBom "org.springframework.cloud:spring-cloud-dependencies:${springCloudVersion}" (6)
	}
}
...
1 引入Spring Boot的Gradle插件。
2 引入Spring依赖管理插件。
3 添加Spring里程碑仓库。
4 统一声明Spring Cloud的版本，Eureka是做为Spring Cloud Netflix项目的一部份编护和发佈的，Spring Cloud的版本也作用于Eureka。
5 将 spring-cloud-starter-netflix-eureka-server 引入至 implementation 范围的依赖。
6 通过 dependencyManagement 统一管理Spring Cloud包的版本。

再然后，在Spring Boot应用配置类上添加注解 @EnableEurekaServer 激活Eureka服务端配置。

EurekaApplication.java
@SpringBootApplication
@EnableEurekaServer (1)
public class EurekaApplication {

	public static void main(String[] args) {
		SpringApplication.run(EurekaApplication.class, args);
	}

}
1 使用注解 org.springframework.cloud.netflix.eureka.server.EnbaleEurekaServer 标注应用入口类，激活Eureka服务端自动配置。

最后，在应用配置文件中配置监听端口及Eureka相关属性。

application-dev.yml
server:
  port: 9001 (1)

eureka:
  instance:
    hostname: localhost (2)
  client:
    registerWithEureka: false (3)
    fetchRegistry: false
    serviceUrl:
      defaultZone: http://${eureka.instance.hostname}:${server.port}/eureka/ (4)
1 设置监听端口为 9001。
2 显示设置服务端主机名。
3 本例Eureka服务端以单独模式运行，所以显示关闭服务注册（以客户端身份）和注册信息同步。
4 将服务端地址指向自身。

在命令行中执行

./gradlew bootRun -Dspring.profiles.active=dev

就会启动Eureka服务端。其提供了一个基于Web的控制面版，在浏覧器地址欗中输入 http://localhost:9001 就可以访问。

注册服务

以 catalog 服务为例。首先，在 build.gradle 中引入Eureka客户端依赖。

build.gradle
...
ext {
	snippetsDir = file('build/generated-snippets')
	set('springCloudVersion', "Greenwich.SR3") (1)
}

dependencies {
    ...
    implementation 'org.springframework.boot:spring-boot-starter-actuator' (2)
	implementation 'org.springframework.cloud:spring-cloud-starter-netflix-eureka-client' (3)
	...
}

dependencyManagement {
	imports {
		mavenBom "org.springframework.cloud:spring-cloud-dependencies:${springCloudVersion}" (4)
	}
}
...
1 跟服务端项目相似，统一设置变量 springCloudVersion 管理Eureka及其它Spring Cloud模组的版本。
2 引入依赖 spring-boot-starter-actuator，以提供状态页 /info 和健康指示 /health。做为服务实例元数据的一部份，服务实例需向Eureka服务端注册状态页和健康指示的URL，而Spring Boot Actuator正好实现了它们。引入 spring-boot-starter-actuator` 是最快的实现符合Eureka规范状态页和健康指示的方法。
3 引入依赖 spring-cloud-start-netflix-eureka-client。
4 统一管理Eureka及其它Spring Cloud模组版本。

然后，使用注解 @EnableEurekaClient 激活Eureka客户端自动配置。

CatalogApplication.java
@SpringBootApplication
@EnableEurekaClient (1)
public class CatalogApplication {

  public static void main(String[] args) {
    SpringApplication.run(CatalogApplication.class, args);
  }
}
1 使用注解 org.springframework.cloud.netflix.eureka.EnableEurekaClient 激活Eureka客户端自动配置。

再然后，在应用配置文件中配置Eureka相关属性。

application-dev.yml
...
eureka:
  client:
    serviceUrl:
      defaultZone: http://localhost:9001/eureka/ (1)
1 本例中，Eureka服务端以单独模式运行，客户端祗需要与一个服务端通信、注册和发现服务。

打开命令行，进入Catalog项目根目录，执行命令：

.gradlew bootRun -Dspring.profiles.active=dev

就可以启动Catalog服务，并向Eureka服务端注册自身。
